# Fix passlib + bcrypt incompatibility

## Root Cause
`passlib` is outdated and doesn't work with `bcrypt>=4.0`. Error: `module 'bcrypt' has no attribute '__about__'`

## Fix
**File:** `skillpath-backend/requirements.txt` (in the separate repo at `C:\Users\stasf\Downloads\skillpath-backend\`)

Change:
```
passlib[bcrypt]>=1.7.4
```
To:
```
passlib[bcrypt]>=1.7.4
bcrypt==3.2.2
```

This pins bcrypt to 3.x which is compatible with passlib.
