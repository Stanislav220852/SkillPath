# Fix Railway — app crashing silently on startup

## Problem
Logs show only Caddy reverse proxy output. No Python/uvicorn output after "Starting Container". App crashes before uvicorn starts.

## Most likely cause
The `pydantic import field_validator` change, or some other import error crashes the app silently. stderr might not be visible in Railway logs.

## Fix 1: main.py — add top-level try/except with print
Add explicit error printing at the very top of main.py so ANY import error is visible:

```python
import sys
import traceback
try:
    # ... existing code ...
except Exception as e:
    print(f"FATAL STARTUP ERROR: {e}", flush=True)
    traceback.print_exc()
    sys.exit(1)
```

## Fix 2: Dockerfile — ensure Python outputs are unbuffered
Add `ENV PYTHONUNBUFFERED=1` to Dockerfile so all print/log output is visible in Railway logs immediately.

## Fix 3: Verify pydantic version compatibility
The `field_validator` with `mode="before"` requires pydantic>=2.0. Check if installed version supports it.
