# Fix Railway Deploy — DATABASE_URL format

## Root Cause
Railway PostgreSQL gives `DATABASE_URL=postgresql://...` but asyncpg needs `postgresql+asyncpg://...`. App crashes on first DB connection.

## Fix 1: config.py — auto-convert URL scheme
**File:** `src/app/skillpath-backend/src/core/config.py`

Add a validator that converts `postgresql://` to `postgresql+asyncpg://` automatically:

```python
@validator("DATABASE_URL", pre=True)
def fix_database_url(cls, v):
    if v and v.startswith("postgresql://"):
        return v.replace("postgresql://", "postgresql+asyncpg://", 1)
    return v
```

## Fix 2: Railway — set DATABASE_URL on backend service
In Railway dashboard → backend service → Variables:
- Add `DATABASE_URL` = `${{PostgreSQL.DATABASE_URL}}` (variable reference to the PostgreSQL plugin)
