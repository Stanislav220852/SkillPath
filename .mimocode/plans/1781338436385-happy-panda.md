# Deploy Backend to Railway + Connect Frontend

## Problem
Frontend: `https://skillpath-mary-and-stas-eta.vercel.app/`
Backend: `localhost:8000` (only local)
Need to deploy backend to Railway and connect them.

## Step 1: Frontend — Make API URL configurable

### `src/app/api.ts` (line 3)
```ts
// Before:
const API_URL = "http://localhost:8000";
// After:
const API_URL = import.meta.env.VITE_API_URL || "http://localhost:8000";
```

### `src/app/components/pages/ProfilePage.tsx`
- Line ~312: hardcoded `fetch("http://localhost:8000/api/auth/me", ...)` → use `API_URL` from api.ts
- Line ~592: `http://localhost:8000${avatarUrl}` → use `API_URL` from api.ts

### Create `.env` in project root
```
VITE_API_URL=http://localhost:8000
```

### Create `.env.production` in project root
```
VITE_API_URL=
```
(User sets real Railway URL in Vercel dashboard, not in git)

### Export API_URL from api.ts
Add `export { API_URL };` so ProfilePage can import it.

## Step 2: Backend — Railway deploy

Backend already has Dockerfile and requirements.txt. Railway will detect and build automatically.

### Environment variables to set in Railway:
| Variable | Value |
|---|---|
| `DATABASE_URL` | Auto-generated by Railway PostgreSQL plugin |
| `JWT_SECRET` | Random secret (generate new one) |
| `JWT_ALGORITHM` | HS256 |
| `JWT_EXPIRE_MINUTES` | 1440 |
| `CORS_ORIGINS` | `https://skillpath-mary-and-stas-eta.vercel.app` |

### Steps:
1. Go to railway.app → Create new project
2. "Deploy from Dockerfile" pointing to `src/app/skillpath-backend/`
3. Add PostgreSQL plugin → it auto-sets `DATABASE_URL`
4. Set env vars above in Variables tab
5. Railway generates URL like `https://xxx.up.railway.app`
6. Test: open `https://xxx.up.railway.app/health` → should return `{"status": "ok"}`

## Step 3: Vercel — Set production env var

In Vercel dashboard → Settings → Environment Variables:
| Key | Value | Environment |
|---|---|---|
| `VITE_API_URL` | `https://xxx.up.railway.app` | Production |

Then redeploy frontend.
