# Plan: Fix Admin Panel Layout, Buttons & UX

## Problem Summary
Admin panel has 3 categories of issues:
1. **Layout**: Pages render full-screen but hide under the fixed navbar (no padding-top, z-index collision)
2. **Buttons**: "Back" and "Save" buttons appear broken due to silent error swallowing and illogical navigation flow
3. **UX**: No shared layout, duplicated headers, confusing back-navigation (SubPage ‚Üí Dashboard ‚Üí Home instead of SubPage ‚Üí Home)

## Root Causes
- Main navbar: `fixed top-0 z-50` (App.tsx:1412) ‚Äî always visible, always on top
- Admin pages: `sticky top-0 z-50` headers (e.g., AdminDashboard.tsx:32) ‚Äî same z-index, positioned at top-0, hidden behind navbar
- Admin pages have no `padding-top` to offset for the ~80px navbar
- Each admin page duplicates its own header/nav bar (no shared layout)
- Empty `catch {}` in API calls (AdminSettings.tsx:43, AdminMentors.tsx:65, etc.) ‚Äî errors silently swallowed
- "‚Üê Back" on sub-pages navigates to AdminDashboard, not to site

## Implementation Plan

### Step 1: Hide main Navbar on admin pages
**File: `src/app/App.tsx`**
- In the Content component, conditionally render Navbar only when NOT on admin pages
- Change from always rendering `<Navbar>` to `{!currentPage.startsWith("admin") && <Navbar>}`
- This eliminates the z-index collision and overlap entirely

### Step 2: Create shared AdminLayout wrapper component
**New file: `src/app/components/pages/AdminLayout.tsx`**
- Wraps all admin pages with a consistent layout
- Contains:
  - Top bar: SkillPath logo ‚Üí "‚Üê Back to Site" (goes to "home") ‚Üí "Logout" button
  - Left sidebar (desktop) / horizontal tabs (mobile): Dashboard, Users, Mentors, Bookings, Settings
  - Active tab highlighted
  - Content area below
- Props: `children`, `activeTab`, `onNavigate`, `onLogout`
- Replaces the duplicated header in each admin page

### Step 3: Refactor all admin pages to use AdminLayout
**Files: AdminDashboard.tsx, AdminUsers.tsx, AdminMentors.tsx, AdminBookings.tsx, AdminSettings.tsx**
- Remove the duplicated `sticky top-0 z-50` header from each page
- Wrap each page's content in `<AdminLayout activeTab={...} onNavigate={...} onLogout={...}>`
- Keep only the page-specific content (tables, forms, stats)

### Step 4: Fix "Save" button error handling
**File: `src/app/components/pages/AdminSettings.tsx`**
- Replace `catch {}` with `catch (e) { alert("–û—à–∏–±–∫–∞ —Å–æ—Ö—Ä–∞–Ω–µ–Ω–∏—è: " + (e?.message || "–ù–µ–∏–∑–≤–µ—Å—Ç–Ω–∞—è –æ—à–∏–±–∫–∞")); }`
- Show visual error state (red border/text) when save fails
- Same pattern for AdminMentors (handleSubmit catch), AdminUsers (handleDelete catch)

### Step 5: Improve "Back" navigation UX
**In AdminLayout:**
- "‚Üê Back to Site" always visible in top bar ‚Üí goes to `home`
- Sidebar tabs allow direct navigation between admin sections (no need to go through Dashboard)
- Each sub-page can directly access any other admin section

### Step 6: Add error feedback across all admin pages
**Files: AdminUsers.tsx, AdminMentors.tsx, AdminBookings.tsx**
- Replace empty `catch {}` blocks with error state display
- Add `error` state + inline error message when API calls fail
